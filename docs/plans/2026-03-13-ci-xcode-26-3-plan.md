# CI Xcode 26.3 Alignment Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Update the release GitHub Actions workflow so the DMG build uses Xcode 26.3 instead of Xcode 16.2.

**Architecture:** Keep the existing `macos-15` runner and the existing package/sign/notarize pipeline. Change only the explicit Xcode selection and add an explicit version log step so the build remains deterministic and auditable.

**Tech Stack:** GitHub Actions, Xcode, xcodebuild, bash

---

### Task 1: Update Workflow Xcode Selection

**Files:**
- Modify: `/.github/workflows/release-app.yml`

**Step 1: Inspect the current workflow**

Open the workflow and confirm it currently uses:

- `runs-on: macos-15`
- `sudo xcode-select -s /Applications/Xcode_16.2.app/Contents/Developer`

**Step 2: Change the selected Xcode version**

Update the workflow step label and the selected developer directory to `Xcode 26.3`:

```yaml
- name: Select Xcode 26.3
  run: sudo xcode-select -s /Applications/Xcode_26.3.app/Contents/Developer
```

**Step 3: Add an explicit verification step**

Immediately after the selection step, add:

```yaml
- name: Verify Xcode version
  run: xcodebuild -version
```

This ensures the effective Xcode version is visible in the workflow logs.

**Step 4: Review the workflow diff**

Confirm no other release behavior changed:

- signing setup is unchanged
- packaging script invocation is unchanged
- notarization flow is unchanged

**Step 5: Commit**

```bash
git add ".github/workflows/release-app.yml"
git commit -m "ci: align release workflow with Xcode 26.3"
```

### Task 2: Validate Release Build Behavior

**Files:**
- Inspect: `/.github/workflows/release-app.yml`
- Inspect: `/scripts/build/package-app.sh`

**Step 1: Validate local assumptions**

Confirm the packaging script does not hardcode a different Xcode path and relies on the selected developer directory from the workflow.

**Step 2: Trigger or run the release workflow**

Use either:

```bash
gh workflow run release-app.yml
```

or push a release tag that matches the workflow trigger.

**Step 3: Check workflow logs**

Verify the log contains:

```text
Xcode 26.3
Build version 17C529
```

**Step 4: Compare UI behavior**

Install the CI-produced DMG and compare the settings window to the app launched from local Xcode 26.3. Confirm the system control styling is now aligned as expected.

**Step 5: Commit follow-up changes if needed**

If no additional changes are required, no follow-up code change is needed. If CI exposes unrelated build issues, address them in a separate commit rather than broadening the workflow change.
